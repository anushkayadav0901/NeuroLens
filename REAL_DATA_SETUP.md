# Using Real BraTS Dataset - Setup Guide

## Current Status
❌ You're using FAKE/SAMPLE data generated by `create_sample_nifti.py`
✅ You want to use REAL BraTS dataset

## What You Need

### 1. Download Real BraTS Data

**Option A: Kaggle (Easiest)**
```bash
# Install Kaggle CLI
pip install kaggle

# Download BraTS dataset
kaggle datasets download -d dschettler8845/brats-2021-task1
```

**Option B: Official BraTS Challenge**
- Visit: https://www.synapse.org/#!Synapse:syn25829067
- Register and download dataset
- Extract files

**Option C: Hugging Face**
- Visit: https://huggingface.co/datasets/Spirit-26/BraTS-2024-Complete
- Download sample cases

### 2. BraTS File Structure

Each patient case typically has 4 MRI sequences + 1 segmentation:

```
BraTS2021_00001/
├── BraTS2021_00001_flair.nii.gz    ← FLAIR (we need this)
├── BraTS2021_00001_t1.nii.gz       ← T1 (optional)
├── BraTS2021_00001_t1ce.nii.gz     ← T1 contrast (optional)
├── BraTS2021_00001_t2.nii.gz       ← T2 (optional)
└── BraTS2021_00001_seg.nii.gz      ← Segmentation (we need this)
```

### 3. Place Files in Your Project

**Copy the files to:**
```
data/BraTS/
├── flair.nii.gz  ← Copy from BraTS2021_XXXXX_flair.nii.gz
└── seg.nii.gz    ← Copy from BraTS2021_XXXXX_seg.nii.gz
```

**Windows Command:**
```bash
# Navigate to your downloaded BraTS folder
cd path/to/BraTS2021_00001

# Copy to your project
copy BraTS2021_00001_flair.nii.gz C:\path\to\your\project\data\BraTS\flair.nii.gz
copy BraTS2021_00001_seg.nii.gz C:\path\to\your\project\data\BraTS\seg.nii.gz
```

### 4. Generate Real MRI Slices

Once you have real files in `data/BraTS/`:

```bash
# Install dependencies (if not already)
pip install nibabel numpy pillow

# Generate slices
python src/krishmodel/backend/generate_mri_slices.py
```

This will:
- Read real BraTS NIfTI files
- Extract 64+ axial slices
- Overlay real tumor segmentation
- Save to `public/mri_slices/`

### 5. Verify Output

Check that real slices were generated:
```
public/mri_slices/
├── metadata.json
├── slice_000.png
├── slice_001.png
└── ... (should show REAL brain scans)
```

### 6. View in App

```bash
npm run dev
```

Navigate to Doctor Dashboard → Click "2D Slices"

You should now see REAL MRI brain scans with actual tumor segmentation!

## Differences: Fake vs Real Data

### Fake Data (Current)
- Generated by `create_sample_nifti.py`
- Synthetic brain shape (sphere)
- Fake tumor (small blob)
- Not realistic

### Real Data (After Setup)
- Actual patient MRI scans
- Real brain anatomy
- Real tumor segmentation
- Medical-grade quality

## Multiple Cases (Advanced)

To support multiple patients:

```
data/BraTS/
├── case_001/
│   ├── flair.nii.gz
│   └── seg.nii.gz
├── case_002/
│   ├── flair.nii.gz
│   └── seg.nii.gz
└── case_003/
    ├── flair.nii.gz
    └── seg.nii.gz
```

Then update `generate_mri_slices.py` to process multiple cases.

## Troubleshooting

### "File not found" error
- Make sure files are in `data/BraTS/`
- Check file names match exactly: `flair.nii.gz` and `seg.nii.gz`

### "nibabel not installed"
```bash
pip install nibabel
```

### Slices look wrong
- Try different axis: Change `axis=2` to `axis=0` or `axis=1`
- Check if segmentation matches MRI dimensions

### No tumor visible
- Verify `seg.nii.gz` has non-zero values
- Try adjusting overlay alpha in script
